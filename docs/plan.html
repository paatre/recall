Phase 1: Quick Wins

Streamline Your Manual Process

The lowest-hanging fruit is to make your current data gathering process faster. Instead of building new tools, this phase focuses on using your existing tools more effectively and creating simple scripts to automate the data extraction. This is your foundation for immediate relief.

Firefox History

Use advanced search filters to narrow history by date range. A simple Python script can directly query the `places.sqlite` file for a more structured output of your browsing activity.

GitLab Activity

Utilize the GitLab API. A personal access token and a short script can pull all your contributions (commits, MRs, comments) within a specific date range into a clean text file.

Slack Search

Master Slack's advanced search. Use operators like `from:me` and `during:yyyy-mm-dd` to instantly find your conversations and contributions for any given day.

Calendar Events

The Google Calendar API can easily export your daily schedule. This provides the backbone of your day, outlining meetings and dedicated work blocks.

Phase 2: Automated Tracking

Evaluate Existing Tools

Before building from scratch, explore existing applications that automate activity tracking. These tools run in the background, passively logging your digital footprint, which could eliminate most of your manual data gathering. Your concern about ActivityWatch on Wayland is valid, but the situation is improving and it's worth a try.

ActivityWatch

The best open-source, self-hosted option. Tracks active windows, browser tabs, and code editor activity. It's highly extensible.

Wayland Status:

Compatibility is actively being developed. Some users report success with recent versions or specific configurations. It's worth installing to test on your system as it may work sufficiently for your needs.

ManicTime

A powerful, cross-platform (including Linux) commercial tool. It offers automatic tracking of apps, documents, and browsing, with features for auto-tagging based on rules.

Consideration:

It might be more powerful than you need and has a subscription model, but offers a very polished experience out of the box.

WakaTime / CodeTime

Excellent for developers, these tools live in your IDE and automatically track time spent writing code, right down to the file and language level.

Limitation:

Their focus is narrow. They won't capture non-coding work like research, meetings, or Slack discussions, which are key parts of your context.

Phase 3: Custom Solution

Build a Personal Aggregator

If existing tools don't fit, build your own. This gives you complete control. The core idea is a modular system with collectors for each data source, a central database to store events, and a simple web UI to view your daily timeline. This is your single source of truth for "what did I work on?".

Phase 4: Full Automation

Integrate AI and Auto-Submission

This is the ultimate goal. Leverage your custom aggregator from Phase 3, add a layer of intelligence to categorize your activity, and finally, automate the submission to your company's time tracking tool. This phase turns your data into action, virtually eliminating the manual logging process.

1. Project & Task Syncing

Use your tool's GraphQL API to fetch a list of your assigned projects and tasks. This provides the 'ground truth' for what you can book time against.

2. AI-Powered Suggestions

Feed the day's aggregated activity and the task list into an LLM. With a well-crafted prompt, it can group activities, suggest time allocations for each task, and generate summaries.

3. "One-Click" Approval Workflow

Your UI will present the AI's suggestions in a clear format. You simply review, make any minor adjustments, and click "Submit".
